### 4. Menu Selection Bug (2:00-4:00 PM)
**Issue:** User could select apps but "Ready to be Still" button wouldn't trigger still mode

**Root cause identified by agent:** Menu closing race condition
- When user clicked an app, menu would rebuild and close/reopen
- This interrupted the event chain
- Next click on "Ready to be Still" wouldn't register

**Fix implemented:** Update menu items in-place
- Removed `buildAndShowMenu()` rebuild after selection
- Now updates checkmarks and button state directly
- Menu stays open for immediate "Ready to be Still" click

## Lessons Learned

- Menu event handling is tricky; rebuilding mid-interaction breaks event chains
- Menubar apps are invisible to AppleScript UI automation (security feature)
- Bundle IDs are more reliable than app object references
- Keychain is the right way to store secrets (not NSUserDefaults)
