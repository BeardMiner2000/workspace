# Session Log — March 26, 2026

## Summary
Formalized and extended a chat-memory system so conversation context survives thread/session loss and can be recovered logically.

## Decisions
- Treat workspace files as durable memory, not chat threads.
- Add `memory/chat-index.md` as a compact cheat sheet/index of notable conversations and workstreams.
- Add `scripts/session_memory.py` to search, summarize, auto-tag, and index historical session transcripts from `~/.openclaw/agents/main/sessions/`.
- Add `scripts/project_context.sh` / `scripts/project_resume.sh` as quick helpers for resuming projects from prior chat context.
- Add `memory/projects/` for project-specific rollups.
- Update startup/process docs so future work checks project memory, chat index, and historical session logs when a task references prior work or missing context.
- Add a recurring cron reminder every 4 hours to trigger chat-memory rollups in the main session.
- Add a selective LLM-summary path for only fresh/uncataloged sessions, with local indexing remaining the default first pass.

## Current Limits
- Local summaries/tags are working now.
- Project-memory scaffolding is in place.
- The LLM summary path is implemented but still needs one more refinement because direct `openclaw agent --local` hit session-locking behavior when invoked from inside the active session.
- Best next fix: route transcript summarization through an isolated run/subagent path rather than the active main-session lane.
