## Summary
Formalized and extended a chat-memory system so conversation context survives thread/session loss and can be recovered logically.

## Decisions
- Treat workspace files as durable memory, not chat threads.
- Add `memory/chat-index.md` as a compact cheat sheet/index of notable conversations and workstreams.
- Add `scripts/session_memory.py` to search, summarize, auto-tag, and index historical session transcripts from `~/.openclaw/agents/main/sessions/`.
- Add `scripts/project_context.sh` / `scripts/project_resume.sh` as quick helpers for resuming projects from prior chat context.
- Add `memory/projects/` for project-specific rollups.
- Update startup/process docs so future work checks project memory, chat index, and historical session logs when a task references prior work or missing context.
- Add a recurring cron reminder every 4 hours to trigger chat-memory rollups in the main session.
- Add a selective LLM-summary path for only fresh/uncataloged sessions, with local indexing remaining the default first pass.

## Current Limits
- Local summaries/tags are working now.
- Project-memory scaffolding is in place.
- Direct `openclaw agent --local`/main-lane transcript summarization hit session-locking behavior when invoked from inside the active session.
- Isolated subagent summarization was successfully tested and is the correct path for higher-quality structured chat summaries.
- Integration is now in place at the workflow level: isolated LLM summary JSON is being written into `memory/chat-summaries-llm/`, and `memory/projects/` can be rebuilt directly from those files.
- Added pace-based chat rotation policy across OpenClaw chats, with heuristic state in `memory/chat-pace-state.json` and helper script `scripts/chat_pace.py`.
- Added `scripts/chat_pace.py assess` so the system can automatically recommend stay vs codify vs rotate based on elapsed time, pace classification, project switches, and recovery/decision signals.
- Added `scripts/chat_rotate.py` to perform a rotation checkpoint: rebuild indexes/project memory, mark codified, reset pace state, write a rotation note to daily memory, and produce a user-facing summary of saved projects.
- Added `scripts/rotate_notice.py` to generate a human-friendly rotate notice with suggested fresh-chat openers based on recently saved project topics.
- Added `memory/session-registry.json` + `scripts/work_chat.py` as a friendly work-chat manager so JL can think in projects/work chats rather than raw OpenClaw session entries.
- Remaining improvement is ergonomic: make heartbeat-triggered runs automatically spawn isolated summarizers for fresh sessions without manual orchestration from the main assistant turn.
